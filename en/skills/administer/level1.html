ADMINISTER - Level 1

Critical Learning (AC13): Simple system administration
First year - Semesters 1 and 2

Back to skills
Level definition

Level 1 of the ADMINISTER competency corresponds to simple system administration. This competency represents the first level of learning (Critical Learning AC13). At this level, I am able to configure basic servers (simple computer servers), manage users and permissions (access rights), and perform backups (security copies of data).

This level represents my first steps in system administration, where I learned the fundamental concepts of server management and service configuration. These skills were developed during semesters 1 and 2 (first year of BUT Computer Science) through educational resources R1.03 and R1.04, as well as academic projects SAÉ 1.03 and SAÉ 2.03.

Critical Learning (AC) for Level 1

Here are the 3 critical learning items for Level 1:

AC13.01
Install and configure a development workstation
Configuration of development environments (IDE - Integrated Development Environment, development tools), installation of necessary tools, basic settings (initial configuration)
AC13.02
Install software necessary for activity
Software installation (computer programs), dependency management (software needed to run other software), tool configuration
AC13.03
Configure a simple work environment
Setup of basic environments (simple work environments), initial configuration (starting settings), simple settings (elementary configuration)
Resources studied in class (S1-S2)
R1.03: Introduction to computer architecture
STUDIED
Technologies: C language (system programming language)
Content: Understanding hardware architecture (physical components of a computer), computer components (processor, memory, storage), basic operation. System programming in C (hardware-close programming).
R1.04: Introduction to operating systems and their operation
STUDIED
Technologies: Oracle VM VirtualBox (virtualization software), Linux (operating system)
Content: Operating system principles (software that manages computer resources), process management (running programs), files and permissions (file access rights), basic administration. Virtualization with Oracle VM VirtualBox (creating virtual machines), Linux operating systems (using command line).
Topics covered

• Installation and configuration of a development workstation: IDE, tools, environment
• System programming in C: process management, files, permissions
• Virtualization: Oracle VM VirtualBox, creation and management of virtual machines
• Linux operating systems: process management, files and permissions, simple work environment configuration

Reference knowledge

• Hardware architecture: CPU, memory, storage, peripherals
• Operating systems: processes, threads, files, permissions
• Process management: creation, synchronization, communication
• Files and permissions: access rights, basic security

Traces and contextualized evidence

Here are my traces and evidence for Level 1:

EVIDENCE 1: Virtual machine on Oracle VM VirtualBox

Context: Academic project carried out in the first year (Semester 1) as part of SAÉ 1.03. This involved installing and configuring a virtual machine (simulated computer) on Oracle VM VirtualBox, with a Linux operating system, and installing web services (PHP) on it.

Trace:

  • Virtual machine configuration with Oracle VM VirtualBox
  • Linux installation and configuration on virtual machines
  • Web services (PHP) installation on virtual machines
  • Configuration and installation procedure documentation
  • Configuration files and scripts available on GitHub (repository all-my-projects)

Justification: This project taught me to install and configure a development workstation (AC13.01) by creating a development environment with a virtual machine. I discovered how to install necessary software (AC13.02) by installing Linux and web services on the virtual machine. I also learned to configure a simple work environment (AC13.03) by setting up a Linux environment that worked correctly. I used knowledge from resources R1.03 (C language to understand architecture) and R1.04 (VM and Linux).

Reflection: This project helped me understand the basics of virtualization and system administration. The main difficulty was understanding how to properly configure a virtual machine and install Linux on it. I overcame this difficulty by following tutorials, practicing regularly, and applying concepts taught in resources R1.03 and R1.04. This experience showed me the importance of virtualization for creating isolated and reproducible development environments.

EVIDENCE 2: Linux on Raspberry Pi

Context: Academic project carried out in the first year (Semester 2) as part of SAÉ 2.03. This involved installing and configuring Linux on a Raspberry Pi (mini-computer), configuring the system environment, and administering this embedded system.

Trace:

  • Linux installation on Raspberry Pi
  • System environment configuration (command line, process management)
  • Basic system administration (user management, permissions, files)
  • Installation and configuration procedure documentation
  • Configuration files available on GitHub (repository all-my-projects)

Justification: This project taught me to install and configure a development workstation (AC13.01) by installing Linux on a Raspberry Pi and configuring the environment so it worked correctly. I also discovered how to install necessary software (AC13.02) by installing tools and software I needed on the Linux system. I learned to configure a simple work environment (AC13.03) by setting up a functional Linux environment on this embedded system. I used resource R1.04 (Linux operating system).

Reflection: This project was enriching as it allowed me to work on a real embedded system (Raspberry Pi) rather than a virtual machine. The main difficulty was understanding how to use the Linux command line to administer the system. I overcame this difficulty by studying basic Linux commands, practicing regularly, and applying concepts taught in resource R1.04. This experience showed me the importance of system administration and prepared me to work on Linux systems in production.

Pedagogical reflection and progression
Challenges encountered
The main challenges at this level concerned understanding virtualization, using the Linux command line, and configuring system environments.
Solutions provided
To overcome these difficulties, I studied tool documentation (Oracle VM VirtualBox, Linux), practiced regularly with Linux commands, and applied methods taught in class.

Progression These first experiences were fundamental for developing my understanding of system administration. They prepared me to approach Level 2 with complex system administration, service deployment, and advanced virtualization with Docker.
