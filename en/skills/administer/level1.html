Critical Learning (AC13): Simple system administration
First year - Semesters 1 and 2

Level definition

Level 1 of the ADMINISTER competency corresponds to simple system administration. This competency represents the first level of learning (Critical Learning AC13). At this level, I am able to configure basic servers (simple computer servers), manage users and permissions (access rights), and perform backups (security copies of data).

This level represents my first steps in system administration, where I learned the fundamental concepts of server management and service configuration. These skills were developed during semesters 1 and 2 (first year of BUT Computer Science) through educational resources R1.03, R1.04, R1.10, R1.11, R2.04, R2.05, R2.12 and R2.13, as well as academic projects SAÉ 1.03 and SAÉ 2.03.

Essential Components (CE)

To act with quality in this competency, I must:
• CE4.01 | Respect regulations on privacy and personal data protection
• CE4.02 | Respect the economic, societal and ecological issues of data storage use, as well as different infrastructures (data centers, cloud, etc.)
• CE4.03 | Rely on mathematical foundations
• CE4.05 | Ensure coherence and quality

Professional Situations

This competency is exercised in the following professional situations:
• Deploy a new technical architecture
• Improve an existing infrastructure
• Secure applications and services

Critical Learning (AC) for Level 1

Here are the 3 critical learning items for Level 1:

AC13.01
Identify the different components of a digital system
Recognition of hardware components (processor, memory, storage, peripherals), understanding of system architecture, identification of elements of a computer system

Use basic features of a multitasking system
Process management (running programs), use of multitasking features (simultaneous execution of multiple programs), system resource management

Install and configure a workstation
Operating system installation, workstation configuration, basic settings, setup of a functional work environment

Traces and contextualized evidence

Here are my traces and evidence for Level 1:

EVIDENCE 1: Virtual machine on Oracle VM VirtualBox

Context: Academic project carried out in the first year (Semester 1) as part of SAÉ 1.03. This involved installing and configuring a virtual machine on Oracle VM VirtualBox with the following specifications: Debian 11 operating system (Debian GUI), 60 GB hard drive, and installation of the Monogame library for development.

Trace:

  • Virtual machine configuration with Oracle VM VirtualBox
  • Installation and configuration of Debian 11 (Debian GUI) on the virtual machine
  • Configuration of a 60 GB hard drive
  • Installation of the Monogame library
  • Complete installation guide in PDF documenting all steps
  • Configuration and installation procedure documentation

Justification: This project taught me to identify the different components of a digital system (AC13.01) by configuring a virtual machine with the required specifications (Debian 11, 60 GB hard drive). I discovered how to use basic features of a multitasking system (AC13.02) by using bash commands to navigate and manage the Debian system. I also learned to install and configure a workstation (AC13.03) by installing Debian 11 with its graphical interface, configuring the hard drive, and installing the Monogame library for development. I used knowledge from resources R1.03 (Linux and bash commands) and R1.04 (VM and Linux).

Reflection: This project helped me understand the basics of virtualization and system administration. The main difficulty was understanding how to properly configure a virtual machine and install Linux on it. I overcame this difficulty by following tutorials, practicing regularly, and applying concepts taught in resources R1.03 and R1.04. This experience showed me the importance of virtualization for creating isolated and reproducible development environments.

EVIDENCE 2: Linux Installation and Configuration on Raspberry Pi (SAÉ 2.03)

Context: Academic project carried out in the first year (Semester 2) as part of SAÉ 2.03 "Network Services Installation". This involved installing and configuring Linux on a Raspberry Pi (mini-computer), configuring the system environment, and administering this embedded system. This project was carried out in collaboration with Baptiste CREPON and Tanguy ABDOULVAID.

Trace:

  • Installation of Raspberry Pi OS (Debian-based operating system) on Raspberry Pi
  • Configuration of a fixed IP address via the /etc/dhcpcd.conf file
  • PuTTY configuration for SSH connection to the Raspberry Pi
  • System environment configuration (command line, process management)
  • Basic system administration (user management, permissions, files)
  • Complete installation and configuration guide in PDF documenting all steps (OS installation, fixed IP configuration, SSH connection, network services installation)
  • Installation and configuration procedure documentation

Justification: This project taught me to identify the different components of a digital system (AC13.01) by working with a Raspberry Pi and understanding its hardware architecture (processor, memory, storage). I discovered how to use basic multitasking system features (AC13.02) by using the Linux command line to manage processes and navigate the file system. I learned to install and configure a workstation (AC13.03) by installing Raspberry Pi OS, configuring the fixed IP address, and setting up a functional Linux environment on this embedded system. I used knowledge from resources R1.03 (Linux and bash commands) and R1.04 (Linux operating system).

Reflection: This project was enriching as it allowed me to work on a real embedded system (Raspberry Pi) rather than a virtual machine. The main difficulty was understanding how to correctly configure the fixed IP address and establish the SSH connection with PuTTY. I overcame this difficulty by studying the Raspberry Pi documentation, following the installation guide we created, and applying concepts taught in resources R1.03 and R1.04. This experience showed me the importance of system administration and prepared me to work on Linux systems in production. The installation guide we produced demonstrates our ability to document and share our technical knowledge.

Pedagogical reflection and progression
Challenges encountered
The main challenges at this level concerned understanding virtualization, using the Linux command line, and configuring system environments.
Solutions provided
To overcome these difficulties, I studied tool documentation (Oracle VM VirtualBox, Linux), practiced regularly with Linux commands, and applied methods taught in class.

Progression These first experiences were fundamental for developing my understanding of system administration. They prepared me to approach Level 2 with complex system administration, service deployment, and advanced virtualization with Docker.
