Second year - Semesters 3 and 4

Level definition

Level 2 of the ADMINISTER competency corresponds to complex system administration. This competency is part of the 6 competencies of the BUT Computer Science program and represents the second level of learning (Critical Learning AC23). At this level, I am able to deploy applications with Docker (containerization tool), configure CI/CD environments (Continuous Integration / Continuous Deployment), and monitor performance (monitor system performance).

This level represents an evolution in my ability to manage more complex infrastructures, automate deployments, and ensure system monitoring. These skills were developed mainly during semesters 3 and 4 (second year of BUT Computer Science) through educational resources R2.05, R3.05, R3.06 and R4.08, as well as academic projects SAÉ 3.01, SAÉ 4.01 and my professional internship at HEVA.

Status: ACQUIRED

All resources at this level have been studied and competencies have been validated through concrete projects and professional internship experience.

Critical Learning (AC) for Level 2

Critical Learning (AC) are the essential elements I must master to achieve this competency level. Here are the 3 critical learning items for Level 2:

AC23.01
Deploy services in a network architecture
Service deployment (putting applications online), network configuration (setting up connections between servers), distributed architecture (system distributed across multiple servers)
AC23.02
Use servers and virtualization tools
Virtualization (creating virtual machines), containers (Docker containers - isolated environments), orchestration (automated management of multiple containers), server management
AC23.03
Secure services and associated data
Service security (application protection), data protection (encryption, backups), security management (authentication, authorization, firewall)

R3.06: Network architecture
STUDIED
Content: TCP/IP network architecture (Internet communication protocol), routing (data routing on the network), advanced network services (complex services on the network).

Traces and contextualized evidence

Traces are raw documents that testify to my activity. Evidence are these traces contextualized with reflection on learning. Here are my traces and evidence for Level 2:

EVIDENCE 1: SAÉ 3.01 - Clone Uber (Laravel) - Network deployment

Context: Academic project carried out in the second year (Semester 3) as part of SAÉ 3.01. This involved developing a complete web application similar to Uber using Laravel, requiring deployment of services in a network architecture and service security.

Trace:

  • Source code with complete network architecture available on GitHub
  • Network service configuration (PHP, Laravel, database)
  • Service security (authentication, authorization, data protection)
  • Network architecture documentation
  • Detailed project sheet: see project sheet

Justification: This project allowed me to learn to deploy services in a network architecture (AC23.01) by configuring and deploying a Laravel application with several services that had to work together: the web application, the database and authentication services. I also discovered the importance of securing services and data (AC23.03) by implementing authentication, authorization and data protection. I used knowledge from resources R2.05 (PHP Network Services), R3.05 (WebSockets) and R3.06 (Network Architecture).

Reflection: This project allowed me to understand how to deploy a complex application in a network architecture. The main difficulty was understanding how to correctly configure network services and secure the application. I overcame this difficulty by studying Laravel documentation, following security best practices, and applying concepts taught in resources R2.05, R3.05 and R3.06. This project showed me the importance of security in service deployment.

EVIDENCE 2: SAÉ 4.01 - Clone Uber (.NET + Vue.js) - Docker virtualization

Context: Academic project carried out in the second year (Semester 4) as part of SAÉ 4.01. This involved developing a complete web application similar to Uber using .NET Core for the backend and Vue.js for the frontend, with Docker virtualization for deployment.

Trace:

  • Source code with Docker configuration available on GitHub
  • Dockerfiles for backend (.NET Core) and frontend (Vue.js)
  • docker-compose.yml configuration to orchestrate services
  • Docker deployment documentation
  • Detailed project sheet: see project sheet

Justification: This project allowed me to learn to use virtualization tools (AC23.02) by using Docker to containerize the application and orchestrate all services (backend, frontend, database). I discovered how to simplify deployment with containers. I also learned to deploy services in a network architecture (AC23.01) by configuring a distributed architecture with multiple Docker containers that communicate with each other. I used resource R4.08 (Docker Virtualization).

Reflection: This project was a turning point in my system administration learning as it allowed me to master Docker and containerization. The main difficulty was understanding how to create efficient Dockerfiles and how to orchestrate multiple containers with docker-compose. I overcame this difficulty by studying Docker documentation, practicing with examples, and applying concepts taught in resource R4.08. This project showed me the importance of virtualization for simplifying deployment and application management.

I participated in the development of a project management web application using React (frontend) and FastAPI (Python backend), requiring system administration and service deployment.

Trace:

  • Application configuration and deployment in production
  • System administration (server management, environment configuration)
  • Service and data security
  • Screenshots of the application in production (see project sheet)
  • Contributions to source code with documented Git commits

Justification: This internship allowed me to learn to deploy services in a network architecture (AC23.01) in a real professional context, by deploying a React + FastAPI application that was used in production. I discovered the real challenges of enterprise deployment. I also learned to secure services and data (AC23.03) by implementing security in a professional application with real users. This daily experience allowed me to apply my system administration skills in a real context with all production constraints.

Reflection: This internship was an enriching professional experience that allowed me to put into practice all the system administration skills acquired in class in a real context. The main difficulty was understanding how to deploy and administer an application in production with company constraints. I overcame this difficulty by communicating regularly with the team, following company procedures, and applying system administration best practices. This experience showed me the importance of system administration in production and allowed me to develop my confidence as a system administrator.
