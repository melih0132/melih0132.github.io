Second year - Semesters 3 and 4

Level definition

Level 2 of the CONDUCT competency corresponds to apply a project monitoring approach based on business needs of clients and users. This competency represents the second level of learning (Critical Learning AC25). At this level, I am able to formalize business needs (precisely document expectations), define feasibility criteria (assess if the project is feasible), and implement adapted project monitoring (effectively track progress).

This level represents an evolution in my ability to manage more complex projects, use structured methodologies (organized methods like Scrum), and coordinate teams. These skills were developed during semesters 3 and 4 (second year of BUT Computer Science) through educational resources R3.03, R3.04, R3.10, R3.11, R3.12, R3.13, R4.02, R4.09, R4.10 and R4.11, as well as academic projects SAÉ 2.05, SAÉ 3.01, SAÉ 4.01 and my internship at Helvetics Events.

Traces and contextualized evidence

Here are my traces and evidence for Level 2:

EVIDENCE 1: Clone Uber (Laravel) - Agile Methodology (Scrum)

Context: Academic project carried out in the second year (Semester 3) as part of SAÉ 3.01. This was a complex web application development project with Laravel, requiring the application of Agile methodology (Scrum), formalization of needs, and adapted project monitoring.

Trace:

  • Application of Agile methodology (Scrum) with sprints and user stories
  • Formalization of needs with functional specifications
  • Adapted project monitoring with Scrum boards and backlogs
  • Gantt chart for project task planning and monitoring
  • Source code available on GitHub (repository uber-complete-system-laravel)
  • Detailed project sheet (see project sheet)

Justification: This project allowed me to learn to formalize client needs (AC25.01) by creating detailed functional specifications that served as a reference for the entire team. I also discovered how to plan and monitor an IT project (AC25.02) by organizing work into sprints with Agile methodology (Scrum), using user stories, creating a Gantt chart to visualize task planning and monitoring, and tracking progress with boards. I applied what I learned in resource R3.10 (Agile Methodology - Scrum).

Reflection: This project was my first experience with Agile methodology (Scrum). The main difficulty was understanding how to organize work into sprints and how to manage user stories. I overcame this difficulty by studying Scrum methodology, practicing with agile management tools, and applying concepts taught in resource R3.10. This project showed me the importance of the agile approach for managing complex projects and allowed me to develop my ability to coordinate a team.

EVIDENCE 2: INTERNSHIP - Helvetic Events Agency

Context: Professional internship of 3 months (June-August 2024) at Helvetic Events Agency in Geneva, as a Web Developer. I participated in professional project management, web application project management, formalization of needs, and adapted project monitoring in a real professional context.

Trace:

  • Professional project management in a real context
  • Web application project management with formalization of needs
  • Adapted project monitoring with professional tools
  • Web application in production (proprietary, not public)
  • Detailed project sheet (see project sheet)

Justification: This internship allowed me to learn to formalize client needs (AC25.01) in a real professional context, by understanding what clients really expected and documenting their needs clearly. I also discovered how to plan and monitor an IT project (AC25.02) by using the tools the company used to track progress. This experience allowed me to apply my project management skills in a real context with all professional constraints.

Reflection: This internship was an enriching professional experience that allowed me to develop my project management skills in a real context. The main difficulty was understanding how to adapt methods learned in class to real professional constraints. I overcame this difficulty by communicating regularly with the team, following company procedures, and applying project management best practices. This internship showed me the importance of flexibility in project management and allowed me to develop my ability to manage professional projects.

Pedagogical reflection and progression
Challenges encountered
The main challenges at this level concerned precise needs formalization, application of agile methodologies, management of complex projects with multiple technologies, and adaptation to real professional constraints.
Solutions provided
To overcome these difficulties, I studied needs formalization methods, practiced with agile methodologies (Scrum), used planning tools (Gantt), developed my project communication skills, and applied methods taught in class.

Progression Level 2 allowed me to develop my complex project management skills. Through the SAÉ 3.01 project (Clone Uber Laravel) and my internship at Helvetics Events, I learned to formalize business needs, define feasibility criteria, and implement adapted project monitoring with agile methodologies. I also discovered tools like CESIM for business simulation and ODOO for project management and resource planning. These experiences allowed me to develop expertise in complex project management, from needs formalization to team coordination. The internship particularly taught me to adapt methods learned to real professional constraints, thus preparing my entry into the professional world.
