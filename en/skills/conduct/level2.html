Level definition

Level 2 of the CONDUCT competency corresponds to complex project management. This competency is part of the 6 competencies of the BUT Computer Science program and represents the second level of learning (Critical Learning AC25). At this level, I am able to formalize business needs (precisely document expectations), define feasibility criteria (assess if the project is feasible), and implement adapted project monitoring (effectively track progress).

This level represents an evolution in my ability to manage more complex projects, use structured methodologies (organized methods like Scrum), and coordinate teams. These skills were developed mainly during semesters 3 and 4 (second year of BUT Computer Science) through educational resources R2.10, R2.14, R3.10 and R4.07, as well as academic projects SAÉ 2.05, SAÉ 3.01, SAÉ 4.01 and my internship at HEVA.

Status: ACQUIRED

All resources at this level have been studied and competencies have been validated through concrete projects and professional internship experience.

Critical Learning (AC) for Level 2

Critical Learning (AC) are the essential elements I must master to achieve this competency level. Here are the 3 critical learning items for Level 2:

AC25.01
Formalize business needs
Needs formalization (precise documentation of expectations), functional specifications (detailed description of features), requirements documentation (specifications document)
AC25.02
Define feasibility criteria
Feasibility analysis (assessment of the possibility to carry out the project), criteria definition (success conditions), constraint evaluation (technical, economic, organizational limits)
AC25.03
Implement adapted project monitoring
Monitoring setup (monitoring tools and processes), project monitoring (progress surveillance), process adaptation (adjustment according to needs)

Traces and contextualized evidence

Traces are raw documents that testify to my activity. Evidence are these traces contextualized with reflection on learning. Here are my traces and evidence for Level 2:

EVIDENCE 1: SAÉ 2.05 - Specifications document, Project Management Folder, Gantt

Context: Academic project carried out in the second year (Semester 2) as part of SAÉ 2.05. This was a project management project requiring the formalization of business needs, the definition of feasibility criteria, and the implementation of adapted project monitoring with a Gantt chart.

Trace:

  • Specifications document (document describing needs and requirements)
  • Project management folder (project management documentation)
  • Gantt chart (visual project planning)
  • Documentation available on GitHub (repository all-my-projects)

Justification: This project taught me to formalize business needs (AC25.01) by creating a complete specifications document that precisely described what was expected. I also discovered how to define feasibility criteria (AC25.02) by evaluating technical, economic and organizational constraints to know if the project was feasible. Finally, I learned to implement adapted project monitoring (AC25.03) by using a Gantt chart to plan and track progress. I used resource R2.10 (Management, accounting and Gantt).

Reflection: This project was my first in-depth experience in structured project management. The main difficulty was understanding how to formalize needs precisely and create a realistic schedule with the Gantt chart. I overcame this difficulty by studying needs formalization methods, practicing with planning tools, and applying concepts taught in resource R2.10. This project showed me the importance of planning and monitoring in project success.

EVIDENCE 2: SAÉ 3.01 - Clone Uber (Laravel) - Agile Methodology (Scrum)

Context: Academic project carried out in the second year (Semester 3) as part of SAÉ 3.01. This was a complex web application development project with Laravel, requiring the application of Agile methodology (Scrum), formalization of needs, and adapted project monitoring.

Trace:

  • Application of Agile methodology (Scrum) with sprints and user stories
  • Formalization of needs with functional specifications
  • Adapted project monitoring with Scrum boards and backlogs
  • Source code available on GitHub (repository uber-laravel)

Justification: This project allowed me to learn to formalize business needs (AC25.01) by creating detailed functional specifications that served as a reference for the entire team. I also discovered the Agile methodology (Scrum) for project monitoring (AC25.03) by organizing work into sprints, using user stories, and tracking progress with boards. I applied what I learned in resource R3.10 (Agile Methodology - Scrum).

Reflection: This project was my first experience with Agile methodology (Scrum). The main difficulty was understanding how to organize work into sprints and how to manage user stories. I overcame this difficulty by studying Scrum methodology, practicing with agile management tools, and applying concepts taught in resource R3.10. This project showed me the importance of the agile approach for managing complex projects and allowed me to develop my ability to coordinate a team.

EVIDENCE 3: SAÉ 4.01 - Clone Uber (.NET + Vue.js) - Project communication

Context: Academic project carried out in the second year (Semester 4) as part of SAÉ 4.01. This was a complex web application development project with .NET and Vue.js, requiring project communication (trailer, pitch), agile methodologies, and complex project management with multiple technologies.

Trace:

  • Project communication with trailer (presentation video) and pitch (oral presentation)
  • Agile methodologies applied
  • Complex project management with multiple technologies (.NET, Vue.js)
  • Source code available on GitHub (repository uber-vuejs-dotnet)

Justification: This project allowed me to learn to implement adapted project monitoring (AC25.03) by managing a complex project that used several different technologies. I also discovered the importance of project communication by creating a trailer (video) and a pitch (oral presentation) to explain the project convincingly. I used resource R4.07 (PPP - Trailer/Pitch) to learn these communication techniques.

Reflection: This project was the most complex in terms of project management as it combined several technologies (.NET, Vue.js) requiring effective coordination. The main difficulty was managing communication between different parts of the project and creating an effective presentation. I overcame this difficulty by applying agile methodologies, communicating regularly with the team, and using techniques taught in resource R4.07. This project showed me the importance of communication in project management and allowed me to develop my ability to present a project convincingly.

This experience allowed me to apply my project management skills in a real context with all professional constraints.

Reflection: This internship was an enriching professional experience that allowed me to develop my project management skills in a real context. The main difficulty was understanding how to adapt methods learned in class to real professional constraints. I overcame this difficulty by communicating regularly with the team, following company procedures, and applying project management best practices. This internship showed me the importance of flexibility in project management and allowed me to develop my ability to manage professional projects.
