MANAGE - Level 1

Critical Learning (AC14): Design and implement a database from a client specification
First year - Semesters 1 and 2

Back to skills
Level definition

Level 1 of the MANAGE competency corresponds to design and implement a database from a client specification. This competency represents the first level of learning (Critical Learning AC14). At this level, I am able to design relational databases (databases organized in linked tables), use SQL (query language for databases) to manipulate data, and manage data integrity (ensure data consistency and validity).

This level represents my first steps in data management, where I learned the fundamental concepts of relational databases and the SQL language. These skills were developed during semesters 1 and 2 (first year of BUT Computer Science) through educational resources R1.05, R1.06, R1.09, R2.06, R2.08, R2.10 and R2.12, as well as academic projects SAÉ 1.04 and SAÉ 2.01.

Essential Components (CE)

To act with quality in this competency, I must:
• CE5.01 | by communicating effectively with the different stakeholders of a project
• CE5.02 | by respecting legal rules and current standards
• CE5.03 | by raising awareness of ethical, responsible, sustainable and intercultural management
• CE5.04 | by adopting a proactive, creative and critical approach

Critical Learning (AC) for Level 1

Here are the 2 critical learning items for Level 1:

AC14.01
Set up a relational database
Database modeling (creating data schemas), relational design (organization in tables), schema implementation (database structure)
AC14.02
Query a database with an appropriate language
SQL queries (commands to retrieve data), data manipulation (add, modify, delete), database updates (modify existing data)

Traces and contextualized evidence

Here are my traces and evidence for Level 1:

EVIDENCE 1: LPO

Context

Academic project carried out in the first year (Semester 1) as part of SAÉ 1.04. This involved designing and implementing a complete relational database from a requirements document, following modeling steps (MCD, MLD, MPD) and creating necessary SQL scripts.

Trace
  • Complete modeling: MCD (Conceptual Data Model), MLD (Logical Data Model), MPD (Physical Data Model)
  • SQL scripts for database creation
  • SQL queries with joins to query the database
  • Database structure documentation
  • Detailed project sheet (see project sheet)
Justification

This project taught me to design and implement a relational database (AC14.01) starting from a requirements document. I created all the modeling (MCD, MLD, MPD) before moving to implementation with SQL. I also developed my skills to query and update a database (AC14.02) by writing SQL queries with joins to efficiently retrieve and manipulate data. I used knowledge acquired in resource R1.05.

Reflection

This project helped me understand the importance of modeling before implementation. The main difficulty was understanding how to go from MCD to MLD then to MPD, and how to translate modeling into SQL scripts. I overcame this difficulty by studying modeling examples, practicing regularly, and applying concepts taught in resource R1.05. This experience showed me the importance of planning and modeling in data management.

EVIDENCE 2: Dortan App (Relational database)

Context

Academic project carried out in the first year (Semester 2) as part of SAÉ 2.01. This involved developing a reservation management application for the municipality of Dortan, requiring the creation of a relational database integrated into the WPF application.

Trace
  • Relational database with tables and relationships
  • SQL queries to query and update the database
  • Database integration into the WPF application
  • Application source code with data management available on GitHub
  • Detailed project sheet (see project sheet)
Justification

Thanks to this project, I was able to design and implement a relational database (AC14.01) to manage reservations. I created the tables and defined relationships between them so everything was coherent. I also learned to query and update the database (AC14.02) by integrating SQL queries directly into the WPF application to manage all reservation operations. This was the first time I applied my data management skills in a real application project.

Reflection

This project was enriching as it allowed me to integrate data management into a complete application. The main difficulty was understanding how to connect the WPF application to the database and how to manage data operations. I overcame this difficulty by studying ADO.NET documentation (technology to access databases from .NET), practicing with examples, and applying SQL concepts learned in resource R1.05. This experience showed me the importance of data management in real applications.

Pedagogical reflection and progression

Level 1 allowed me to acquire the basics of data management. Through projects like SAÉ 1.04 (Dataset) and Dortan App (SAÉ 2.01), I learned to model databases (MCD, MLD, MPD), use SQL to query and update data, and ensure data integrity.

Challenges encountered

The main challenges at this level concerned understanding relational modeling, creating complex SQL queries with joins, and integrating databases into applications.

Solutions provided

To overcome these difficulties, I studied relational modeling concepts, practiced regularly with SQL, created test databases, and applied methods taught in class.

Progression These first experiences were fundamental for developing my understanding of data management. They prepared me to approach Level 2 with performance optimization, data security, and data visualization with tools like PowerBI.
