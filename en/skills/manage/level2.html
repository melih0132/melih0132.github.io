Critical Learning (AC24): Complex data management
Second year - Semesters 3 and 4

Back to skills
Level definition

Level 2 of the MANAGE competency corresponds to complex data management. This competency represents the second level of learning (Critical Learning AC24). At this level, I am able to optimize query performance (improve execution speed), use analysis tools like Power BI (data visualization tool), and manage large volumes of data (large amounts of data).

This level represents an evolution in my ability to analyze and visualize data, optimize performance, and manage larger volumes of data. These skills were developed mainly during semesters 3 and 4 (second year of BUT Computer Science) through educational resources R2.06, R3.07 and R4.03, as well as academic projects SAÉ 2.04, SAÉ 3.01, SAÉ 4.01 and my professional internship at HEVA.

Critical Learning (AC) for Level 2

Here are the 3 critical learning items for Level 2:

AC24.01
Optimize database performance
Performance optimization (improving query execution speed), indexing (creating indexes to speed up searches), optimized queries (improved SQL queries for better performance)
AC24.02
Ensure data security and integrity
Data security (protection against unauthorized access), referential integrity (ensuring consistency of relationships between tables), data protection (encryption, backups)
AC24.03
Organize data restitution and visualization
Data restitution (data presentation), visualization (creating charts and tables), reports and dashboards (interactive dashboards)
Resources studied in class (S3-S4)
R2.06: Database exploitation (SQL)
STUDIED
Technologies: SQL (query language for databases)
Content: Performance optimization (indexing, execution plan), data security and integrity (authentication, constraints), restitution and visualization (reports, complex queries).
R3.07: Triggers, Procedures, NoSQL
STUDIED
Technologies: Advanced SQL (Triggers - automatic triggers, Stored Procedures), NoSQL (non-relational databases)
Content: Triggers (automatic triggers on events), stored procedures (reusable SQL functions), introduction to NoSQL databases (MongoDB).
R4.03: Quality and beyond relational (MongoDB)
STUDIED
Technologies: MongoDB (NoSQL database)
Content: NoSQL databases, MongoDB (document data structure), data quality, unstructured data management.
Topics covered

• Database performance optimization: indexing, optimized queries
• Data security and integrity assurance: authentication, validation, transactions
• Data restitution and visualization organization: reports, dashboards, analytics

Reference knowledge

• Query optimization: execution plan, indexes, statistics
• Indexing: B-tree, hash index, composite indexes
• Transactions: ACID, isolation, rollback
• Views and stored procedures: abstraction, reusability
• ORM: object-relational mapping, ORM queries
• Analytics: reports, visualization, business intelligence
