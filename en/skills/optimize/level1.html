Level definition

Level 1 of the OPTIMIZE competency corresponds to simple algorithm optimization. This competency represents the first level of learning (Critical Learning AC12). At this level, I understand algorithmic complexity (execution time and memory space required) and I am able to implement basic algorithms and use appropriate data structures (lists, arrays, dictionaries, JSON, CSV with serialization and deserialization).

This level represents my first steps in algorithm analysis and optimization, where I learned the fundamental concepts of algorithmic complexity and the importance of choosing data structures. These skills were developed during semesters 1 and 2 (first year of BUT Computer Science) through educational resources R1.01, R1.06 and R2.07, as well as academic projects SAÉ 1.01, 1.02 and 2.02.

Critical Learning (AC) for Level 1

Here are the 3 critical learning items for Level 1:

AC12.01
Analyze a problem with method
Systematic problem analysis, decomposition of complex problems into smaller problems, identification of sub-problems, structured approach
AC12.02
Compare algorithms for classic problems
Comparison of different algorithmic approaches, analysis of algorithm efficiency, evaluation of algorithmic solutions for common problems
AC12.03
Experiment and analyze program performance
Performance testing, execution time analysis, resource consumption evaluation, experimental validation of algorithmic solutions

Traces and contextualized evidence

Here are my traces and evidence for Level 1:

EVIDENCE 1: Evit'Shuriken

Context

Academic project carried out in the first year (Semester 1) as part of SAÉ 1.01 and 1.02. This involved developing a simple video game using Unity and C#, requiring algorithm implementation for gameplay management (movement, collisions, score).

Trace
  • Source code with implemented algorithms available on GitHub
  • Game screenshots showing algorithmic features (see project sheet)
  • Code comments explaining algorithmic choices
Justification

This project taught me to analyze a problem by breaking it down into sub-problems (AC12.01). I divided game development into several parts: collision management, score calculation and character movement. I also learned to build simple algorithms (AC12.02) to manage all game interactions. I used what I learned in resource R1.01 (C#) to implement these algorithms.

Reflection

This project helped me understand the importance of breaking down a complex problem into simpler sub-problems. The main difficulty was understanding how to structure algorithms to manage collisions and interactions. I overcame this difficulty by studying Unity documentation, practicing regularly, and applying concepts seen in class (R1.01). This experience showed me the importance of algorithmic planning before implementation.

EVIDENCE 2: Production flow modeling with graphs

Context

Academic project carried out in the first year (Semester 2) as part of SAÉ 2.02. This involved using graph theory to model a food production process, representing the different transformation stages (cutting, grinding, cooking, packaging) and the flows from raw materials to finished products.

Trace

Production flow diagram: modeling of processes (cutting, grinding, cooking, packaging) and products (pork, chicken, duck, terrines, pâtés, hams) with quantities in tons

This modeling represents a food production flow where:

  • Processes (cutting, grinding, cooking, packaging) are represented as transformation nodes
  • Products (raw materials and finished products) are represented as storage nodes
  • Arrows indicate material flows with quantities in tons
  • This structure allows analyzing flows, optimizing production, and making strategic decisions
Justification

This project taught me to analyze a problem by breaking it down into sub-problems (AC12.01) by using graph theory to model a complex production system. I discovered how to represent a real industrial process in the form of a directed graph, where each node represents a stage or a product, and each arc represents a material flow with its quantity. I also learned to build simple algorithms (AC12.02) to analyze these flows and calculate the quantities needed at each stage. I put into practice what I learned in resources R1.06 (Discrete mathematics) and R2.07 (Graphs) to create this modeling.

Reflection

This project was a challenge as I had to understand how to use graph theory to model a real production process. The main difficulty was making the connection between abstract mathematical concepts (directed graphs, flows) and their practical application in an industrial context. I overcame this difficulty by analyzing the process step by step, identifying the inputs and outputs of each transformation, and applying concepts taught in resources R1.06 and R2.07. This project showed me the importance of discrete mathematics and graph theory for algorithmic optimization and modeling complex systems.

Pedagogical reflection and progression
Challenges encountered
The main challenges at this level concerned understanding algorithmic complexity (Big O notation), problem modeling with graphs, and choosing appropriate data structures.
Solutions provided
To overcome these difficulties, I studied theoretical concepts of discrete mathematics and graph theory, practiced regularly by implementing algorithms, and applied methods taught in class.

Progression Level 1 allowed me to understand the importance of algorithmic optimization and develop reflection on solution complexity. Through projects like Evit'Shuriken and production flow modeling with graphs, I learned to analyze my code's performance, break down complex problems into sub-problems, and choose the most appropriate data structures. These first experiences were fundamental for developing my understanding of algorithmic optimization. They prepared me to approach Level 2 with more advanced algorithmic techniques and complex data structures.
