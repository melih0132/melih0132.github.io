Level definition

Level 2 of the OPTIMIZE competency corresponds to complex algorithm optimization. This competency represents the second level of learning (Critical Learning AC22). At this level, I am able to design efficient algorithms, optimize database queries, and implement caching techniques (temporary data storage to improve performance).

This level represents an evolution in my ability to analyze and improve application performance, optimize database interactions, and implement caching strategies. These skills were developed during semesters 3 and 4 (second year of BUT Computer Science) through educational resources R3.02 and R4.04, as well as academic projects SAÉ 3.01.

Traces and contextualized evidence

Here are my traces and evidence for Level 2:

EVIDENCE 1: Clone Uber (Algorithmic optimization)

Context: Academic project carried out in the second year (Semester 3) as part of SAÉ 3.01. This involved developing a complete web application similar to Uber using Laravel, requiring algorithmic optimization to efficiently manage distance calculations, route searches, and database interactions.

Trace:

  • Source code with optimized algorithms available on GitHub
  • Optimization of complex SQL queries
  • Optimized distance calculation algorithms
  • Documentation of algorithmic choices and optimizations performed
  • Detailed project sheet (see project sheet)

Justification: This project taught me to select appropriate data structures (AC22.01) by choosing those that were most efficient for storing and accessing data, particularly with database indexing and cache structures. I also deepened my knowledge of advanced algorithmic techniques (AC22.02) by optimizing distance calculation and route search algorithms to make them faster. I used algorithmic reasoning schemas (AC22.03) to analyze performance and find points to optimize. I applied knowledge from resources R3.02 (Python for analysis) and R4.04 (Optimization method).

Reflection: This project was a major challenge in terms of optimization as it required managing large amounts of data (users, trips, distance calculations). The main difficulty was optimizing SQL queries and calculation algorithms to guarantee acceptable response times. I overcame this difficulty by analyzing SQL query execution plans, using appropriate indexing, implementing cache algorithms, and applying optimization methods taught in resource R4.04. This project showed me the importance of algorithmic optimization in real applications and prepared me to work on production systems.
