Level definition

Level 3 of the OPTIMIZE competency corresponds to advanced optimization and performance. This competency represents the third level of learning (Critical Learning AC32), the most advanced level. In my Application Development pathway, only the REALIZE, OPTIMIZE and COLLABORATE competencies are evaluated at level 3 in semesters 5 and 6. At this level, I am able to develop high-performance solutions, optimize distributed systems (systems distributed across several servers), and implement AI/ML techniques (Artificial Intelligence / Machine Learning - Automatic Learning).

This level represents my ability to manage scalability (a system's ability to handle increasing load) and load, optimize complex systems, and implement advanced optimization solutions. These skills are developed mainly during semesters 5 and 6 (third year of BUT Computer Science) through educational resources R5.04, R5.11 and R5.12, as well as academic projects SAÉ 5.01 and my professional work-study experience at Agence Altimax.

Critical Learning (AC) for Level 3

Here are the 3 critical learning items for Level 3:

AC32.01
Anticipate and evaluate performance (time, memory, precision, load)
Predictive performance analysis (predicting performance before implementation), metric evaluation (execution time, memory usage, load scaling - ability to handle increasing load), proactive optimization (optimization before problems appear)
AC32.02
Analyze code and propose optimizations
Code profiling (detailed analysis of execution time of each part of the code), behavior analysis, bottleneck identification (points that slow down the application), optimization justification
AC32.03
Justify algorithmic choices in view of multiple constraints
Selection of specialized libraries, use of dedicated methods according to domain (AI - Artificial Intelligence, imaging, parallelism - simultaneous execution on multiple processors, etc.), domain optimization, justification of choices taking into account multiple constraints (performance, cost, maintainability)

Traces and contextualized evidence

Here are my traces and evidence for Level 3:

EVIDENCE 1: SAÉ 5.01 - SmartHome (AI Classification and IoT optimization)

Context: Academic project carried out in the third year (Semester 5) as part of SAÉ 5.01. This is a complex project developing an Android mobile application with Kotlin, integrating IoT (Internet of Things) via MQTT and Raspberry Pi, and using artificial intelligence for data classification, requiring advanced performance optimization.

Trace:

  • Kotlin application source code with optimized AI algorithms (project in progress, confidential)
  • Optimized IoT architecture with MQTT (communication protocol for IoT) and Raspberry Pi
  • AI integration for classification with performance optimization
  • Performance analysis and optimization documentation

Justification: This project allowed me to learn to anticipate and evaluate performance (AC32.01) by thinking about execution time, memory and load needs for AI and IoT before even starting to code. I also developed my ability to analyze code and propose optimizations (AC32.02) by identifying parts that slowed down IoT data processing and AI algorithm execution. I had to justify my algorithmic choices (AC32.03) by selecting the most appropriate AI algorithms (classification) taking into account both performance and precision. I used resources R5.11 (Optimization method) and R5.12 (AI - Classification).

Reflection: This project is the most complex in terms of optimization as it combines three different domains: mobile development, IoT and artificial intelligence, each with its own performance constraints. The main difficulty was optimizing AI algorithms so they execute quickly on a mobile device while maintaining good classification precision. I overcame this difficulty by using optimization techniques taught in R5.11, analyzing performance with profiling tools, and applying optimized AI methods from R5.12. This project showed me the importance of optimization in systems integrating several complex technologies.

EVIDENCE 2: WORK-STUDY - Agence Altimax (WordPress code optimization)

Context: Professional work-study since September 2025 (4 months, in progress) as a Web Developer at Agence Altimax in Annecy. I develop and maintain websites with WordPress, optimizing code, improving performance, and analyzing bottlenecks to propose optimizations.

Trace:

  • Optimized WordPress websites in production (proprietary, not public)
  • PHP, HTML, CSS, JavaScript code optimizations performed
  • Measured performance improvements (loading time reduction, query optimization)
  • Performance analyses with profiling tools
  • Optimization documentation and justification of algorithmic choices
  • Regular contributions with documented Git commits

Justification: This work-study allowed me to learn to anticipate and evaluate performance (AC32.01) by thinking about the impact of my optimizations on loading time and user experience before implementing them. I also developed my ability to analyze code and propose optimizations (AC32.02) by identifying parts of WordPress code that slowed down sites, using profiling tools to understand where problems were, and proposing concrete solutions. I had to justify my algorithmic choices (AC32.03) by selecting the best optimization techniques taking into account multiple constraints: performance, maintainability and compatibility with WordPress. This daily experience allows me to apply my level 3 skills in a real professional context with all production constraints.

Reflection: This work-study is a continuous professional experience that allows me to develop my advanced optimization skills in a real context. The main difficulty was understanding how to optimize existing WordPress code without breaking functionality, while respecting company constraints. I overcame this difficulty by analyzing code with profiling tools, testing optimizations on development environments, and measuring performance improvements. This experience showed me the importance of continuous optimization in a professional context and allowed me to develop my ability to justify my algorithmic choices in the face of multiple constraints (performance, cost, maintainability, deadlines).
