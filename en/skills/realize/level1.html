REALIZE - Level 1

Critical Learning (AC11): Simple application development
First year - Semesters 1 and 2

Back to skills
Level definition

Level 1 of the REALIZE competency corresponds to simple application development. This competency represents the first level of learning (Critical Learning AC11). At this level, I master the basic concepts of software development and I am able to implement simple features according to specifications.

This level represents my first steps in application development, where I learned to use an integrated development environment (IDE), implement basic features and perform simple unit tests. These skills were developed during semesters 1 and 2 (first year of BUT Computer Science) through educational resources R1.01, R1.02, R2.01, R2.02 and R2.03, as well as academic projects SAÉ 1.01, 1.02 and 2.01.

Critical Learning (AC) for Level 1

Here are the 4 critical learning items for Level 1:

AC11.01
Implement simple designs
Implementation of basic designs, coding simple features according to specifications, using an appropriate programming language (C#, HTML, CSS)
AC11.02
Develop simple designs
Creation of elementary designs, simple code structuring, basic modeling (simple diagrams, basic architecture)
AC11.03
Perform tests and evaluate results
Simple tests, feature validation, specification compliance verification, basic debugging (error correction)
AC11.04
Develop user interfaces
Creation of basic user interfaces, setup of simple UI components, integration of user interactions (buttons, forms, navigation)
Resources studied in class (S1-S2)

Here are the resources I studied to develop this competency at Level 1:

R1.01: Introduction to development
STUDIED
Technologies: C# (programming language), WPF (Windows Presentation Foundation - framework for creating graphical interfaces)
Content: Introduction to fundamental programming concepts (variables, conditions, loops), control structures, functions, and programming basics. Learning basic programming languages and development tools (Visual Studio).
R1.02: Web interface development
STUDIED
Technologies: HTML (web page structure), CSS (formatting and design)
Content: Creation of web user interfaces with HTML and CSS. Learning principles of simple interface design, basic accessibility, and integration of user interactions.
R2.01: Object-oriented development
STUDIED
Technologies: C# with Object-Oriented Programming (OOP)
Content: Introduction to object-oriented programming: classes (data structures), objects (class instances), inheritance (code reuse), encapsulation (data protection), associations and lists. Code structuring with OOP concepts, basic object-oriented modeling.
R2.02: Application development with HMI
STUDIED
Technologies: Unity (game engine), C#
Content: Development of graphical user interfaces (HMI = Human-Machine Interface), event management (clicks, movements), creation of simple UI components, basic ergonomics principles.
R2.03: Development quality
STUDIED
Technologies: Unit Testing
Content: Development best practices, simple unit testing, code quality, maintainability.

Traces and contextualized evidence

Here are my traces and evidence for Level 1:

EVIDENCE 1: Evit'Shuriken

Context

Academic project carried out in the first year (Semester 1) as part of SAÉ 1.01 and 1.02. This involved developing a simple video game using WPF and C#.

Trace
  • Complete project source code available on GitHub
  • Screenshots of the game interface (see project sheet)
  • Project documentation explaining implemented game mechanics
Justification

Thanks to this project, I learned to implement simple designs (AC11.01) by coding the basic features of a game: character movement, collision management and score system. I also developed my skills to create user interfaces (AC11.04) by designing a main menu, game interface and end screen. By commenting my code, I got into the habit of structuring my work clearly and readably.

Reflection

This first project allowed me to understand the basics of programming and using a development environment. The main difficulties encountered concerned collision management and event synchronization. I solved these problems by studying WPF documentation and practicing regularly. This experience was fundamental for developing my confidence as a beginner developer.

EVIDENCE 2: Dortan App

Context

Academic project carried out in the first year (Semester 2) as part of SAÉ 2.01. This involved designing and developing a reservation management application for the municipality of Dortan using C# WPF and a database.

Trace
  • WPF application source code with object-oriented architecture (GitHub)
  • Screenshots of the complete user interface (see project sheet)
  • Class diagrams showing object-oriented structure
  • Relational database with tables and relationships
Justification

This project allowed me to put into practice developing simple designs (AC11.02) by creating an object-oriented architecture with classes, interfaces and inheritance. I also developed a complete user interface (AC11.04) with WPF, including forms, lists and all necessary interactions. By regularly testing my application, I learned to perform tests and evaluate results (AC11.03) to ensure everything worked correctly.

Reflection

This project was a turning point in my learning as it allowed me to apply object-oriented programming concepts seen in class (R2.01). The main difficulty was understanding how to structure code with inheritance and interfaces. I overcame this difficulty by creating class diagrams before coding and following principles taught in class. This project showed me the importance of planning before development.

EVIDENCE 3: KUBE

Context

Academic project carried out in the first year as part of resource R2.02 (Application development with HMI). This involved developing a 3D game with Unity and C#, creating interactive user interfaces and managing events and user interactions.

Trace
  • Complete Unity project source code available on GitHub
  • Screenshots of the game interface (see project sheet)
  • Project documentation explaining implemented game mechanics
Justification

Thanks to this project, I learned to develop user interfaces (AC11.04) by creating a main menu, a 3D game interface and an end screen with Unity. I also put into practice implementing simple designs (AC11.01) by coding the basic features of a 3D game: character movement, obstacle management and navigation system. This project allowed me to discover Unity and understand the principles of application development with HMI.

Reflection

This project allowed me to discover video game development with Unity. The main difficulties encountered concerned 3D physics management, navigation in a 3D environment and event synchronization. I solved these problems by studying Unity documentation and practicing regularly. This experience showed me the importance of understanding development tools and following best practices to create smooth and intuitive user interfaces.

Pedagogical reflection and progression

Level 1 allowed me to acquire the essential foundations of software development according to the BUT Computer Science competency framework. Through concrete projects like Evit'Shuriken (WPF), KUBE (Unity) and Dortan App (WPF), I learned to structure my code, use development tools (Visual Studio, Unity) and implement features according to precise specifications.

Challenges encountered

The main challenges at this level concerned understanding technical specifications, managing code complexity and learning development tools. I also had to learn to debug (correct errors) effectively.

Solutions provided

To overcome these difficulties, I used official documentation of technologies (Microsoft, Unity), practiced regularly by creating personal projects, collaborated with my peers during group work, and applied methods taught in class.

Progression These first experiences were crucial for developing my understanding of basic concepts and my confidence as a developer. They prepared me to approach Level 2 with more complex projects requiring integration of multiple technologies and implementation of advanced testing processes.
