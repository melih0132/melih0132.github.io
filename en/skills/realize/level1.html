REALIZE - Level 1

Critical Learning AC11 : Development of simple applications

Back to skills
Level definition

Level 1 (AC11) of the REALIZE competency corresponds to the development of simple applications. At this level, I master the basic concepts of software development and I am able to implement simple features according to specifications.

This level represents my first steps in application development, where I learned to use an integrated development environment, to implement basic features and perform simple unit tests.

Critical Learnings for Level 1

• AC11.01 | Implement simple designs : Implementation of basic designs, coding simple features according to specifications
• AC11.02 | Develop simple designs : Creation of elementary designs, simple code structuring
• AC11.03 | Conduct tests and evaluate results against specifications : Simple tests, feature validation, specification compliance verification
• AC11.04 | Develop user interfaces : Creation of basic user interfaces, implementation of simple UI components

Resources studied in class (S1-S2)

R1.01 : Introduction to development
Introduction to fundamental programming concepts, control structures, functions, and programming basics.

R1.02 : Web interface development
Creation of web user interfaces with HTML, CSS and basic JavaScript. Learning principles of simple interface design.

R2.01 : Object-oriented development
Introduction to object-oriented programming: classes, objects, inheritance, encapsulation. Code structuring with OOP concepts.

R2.02 : Application development with GUI
Development of graphical user interfaces, event handling, creation of simple UI components.

Topics covered

• Basic programming: variables, data types, control structures (if/else, loops)
• Functions and procedures: modularity, parameters, return values
• Object-oriented programming: classes, objects, methods, properties
• Simple user interfaces: creation of basic UI components, event handling
• Basic HTML/CSS/JavaScript: web structure, styles, simple interactions

Reference knowledge

• Programming languages: syntax, semantics, paradigms
• Data structures: lists, arrays, dictionaries
• Basic algorithms: sorting, searching, data manipulation
• Interface design: basic ergonomics principles, elementary accessibility

Content to complete

To be completed with your personal projects and experiences:

Traces and evidence

Commented source code : All projects are available on GitHub with explanatory comments allowing to understand the implementation of basic features.

Application screenshots : Screenshots of developed interfaces showing the concrete realization of features.

Unit tests : Test files demonstrating the ability to validate code with simple tests.

Technical documentation : Project documentation explaining technical choices and basic architecture.

Pedagogical reflection

Level 1 allowed me to acquire the essential foundations of software development. Through concrete projects like DortanApp and Evit'Shuriken, I learned to structure my code, to use development tools and to implement features according to specifications. These first experiences were crucial to develop my understanding of basic concepts and my confidence as a developer.

The main challenges encountered at this level concerned understanding specifications, code complexity management and tool learning. The solutions provided were the use of documentation, regular practice and collaboration with my peers.
