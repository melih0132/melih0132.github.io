Level definition

Level 2 of the REALIZE competency corresponds to the development of complex applications. This competency represents the second level of learning (Critical Learning AC21). At this level, I am able to design application architecture, integrate multiple technologies and frameworks, and implement testing and deployment processes.

This level represents a significant evolution in my ability to manage technical complexity, integrate different technologies and implement automated processes to guarantee application quality and deployment. These skills were developed during semesters 3 and 4 (second year of BUT Computer Science) through educational resources R3.01, R3.04, R4.01, R4.02 and R4.11, as well as academic projects SAÉ 3.01, SAÉ 4.01 and my professional internship at Helvetics Events.

Critical Learning (AC) for Level 2

Here are the 4 critical learning items for Level 2:

AC21.01
Develop and implement functional and non-functional specifications from requirements
Analysis and formalization of needs (what the application must do), implementation of functional specifications (features) and technical constraints (performance, security, maintainability)
AC21.02
Apply ergonomics and accessibility principles
Design of ergonomic interfaces (easy to use), respect for accessibility standards (for people with disabilities), optimal user experience (UX - User Experience)
AC21.03
Apply good design and programming practices
Design patterns (reusable solutions to solve common problems), SOLID principles (object-oriented programming rules), refactoring (improving code without changing functionality), clean and maintainable code (easy to understand and modify)
AC21.04
Verify and validate an application through testing
Unit tests (tests of each function), integration tests (tests of several components together), validation tests (verification that the application meets needs), quality assurance (guarantee code quality)

Traces and contextualized evidence

Here are my traces and evidence for Level 2:

EVIDENCE 1: Clone Uber (Laravel)

Context

Academic project carried out in the second year (Semester 3) as part of SAÉ 3.01. This involved developing a complete web application similar to Uber using Laravel (PHP framework) with an MVC architecture (Model-View-Controller).

Trace
  • Complete source code with MVC architecture on GitHub
  • UML diagrams (Unified Modeling Language) showing the application architecture
  • Unit tests and integration tests with PHPUnit
  • Technical documentation explaining architectural choices
  • Detailed project sheet (see project sheet)
Justification

This project taught me to analyze needs and transform specifications into functional code (AC21.01). I had to understand user expectations and translate them into concrete features. I also discovered the importance of good design practices (AC21.03) by using MVC architecture, design patterns and SOLID principles to make my code more maintainable. Finally, I implemented automated tests (AC21.04) that allowed me to verify that everything worked as expected.

Reflection

This project was a major challenge as I had to integrate many technologies (Laravel, PHP OOP, database, authentication, payment). The main difficulty was understanding MVC architecture and applying it correctly. I overcame this difficulty by studying Laravel documentation, following tutorials, and practicing regularly. This project showed me the importance of architectural planning before development.

EVIDENCE 2: Clone Uber (.NET + Vue.js)

Context

Academic project carried out in the second year (Semester 4) as part of SAÉ 4.01. This involved developing a complete web application similar to Uber using .NET Core for the backend and Vue.js for the frontend, with a modern architecture and automated testing.

Trace
  • Source code with .NET Core and Vue.js architecture on GitHub
  • Unit tests with mocks (simulations) to isolate tests
  • Docker configuration for deployment
  • API documentation with Swagger (API documentation tool)
  • Detailed project sheet (see project sheet)
Justification

This project allowed me to learn to integrate multiple technologies (AC21.01) by combining .NET Core for the backend and Vue.js for the frontend, creating a true full-stack application. I deepened my knowledge of development best practices (AC21.03) by writing unit tests with mocks, which helped me isolate each part of my code. Thanks to a complete test suite, I was able to verify and validate my application (AC21.04). By working on the Vue.js interface, I also learned to apply ergonomics principles (AC21.02) to make the user experience pleasant.

Reflection

This project allowed me to deepen my software architecture and automated testing skills. The main difficulty was understanding how to use mocks to isolate tests and manage communication between frontend and backend. I solved these problems by studying testing best practices, using tools like Moq (mocking framework for .NET), and practicing regularly. This project prepared me to work on professional applications.

This internship allowed me to work on real specifications (AC21.01) in a professional context, by collaborating with an experienced developer team. I also discovered the importance of automated testing (AC21.04) to guarantee code quality in production. By developing the React interface, I was able to apply ergonomics principles (AC21.02) to create a fluid and intuitive user experience.

This internship was an enriching professional experience that allowed me to put into practice all the skills acquired in class in a real context. The main difficulty was adapting to existing code and team practices. I overcame this difficulty by communicating regularly with the team, studying existing code, and following company best practices. This experience showed me the importance of team collaboration and communication in professional development. It also allowed me to develop my confidence as a developer and understand the real challenges of enterprise development.

The main challenges were managing increasing complexity, integrating multiple technologies (Javascript, PHP OOP, .NET Core, Vue.js, Flutter), and implementing automated testing processes.
Solutions provided
The solutions provided were adopting good architectural practices, using design patterns, and automating testing and deployment processes with unit tests and mocks.

Progression Level 2 was a turning point in my learning, allowing me to move from implementing isolated features to designing complete and integrated applications. The Clone Uber projects (Laravel and .NET + Vue.js) and my internship at Helvetics Events particularly taught me to manage technical complexity, integrate different technologies and think in terms of architecture.
