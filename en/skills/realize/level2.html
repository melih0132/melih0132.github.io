REALIZE - Level 2

Critical Learning AC21 : Development of complex applications

Back to skills
Level definition

Level 2 (AC21) of the REALIZE competency corresponds to the development of complex applications. At this level, I am able to design application architecture, integrate multiple technologies and frameworks, and implement testing and deployment processes.

This level represents a significant evolution in my ability to manage technical complexity, integrate different technologies and implement automated processes to guarantee application quality and deployment.

Critical Learnings for Level 2

• AC21.01 | Develop and implement functional and non-functional specifications from requirements : Analysis and formalization of needs, implementation of functional specifications and technical constraints
• AC21.02 | Apply ergonomics and accessibility principles : Design of ergonomic interfaces, respect for accessibility standards, optimal user experience
• AC21.03 | Apply good design and programming practices : Design patterns, SOLID principles, clean and maintainable code
• AC21.04 | Verify and validate an application through testing : Unit tests, integration tests, validation tests, quality assurance

Resources studied in class (S3-S4)

R2.01 : Object-oriented development
Advanced object-oriented programming: design patterns, SOLID principles, object-oriented architectures, code reusability.

R2.02 : Application development with GUI
Advanced user interface development, ergonomics and accessibility, complex state management, optimal user experience.

R2.03 : Development quality
Development best practices, unit testing, continuous integration, code quality, maintainability.

R3.01 : Web development
Complete web application development, MVC architectures, REST APIs, modern frameworks, separation of concerns.

R3.04 : Development quality
Quality deepening: integration testing, code coverage, code review, quality assurance, validation and verification.

Topics covered

• Advanced object-oriented programming: design patterns, SOLID principles, architectures
• Modern frameworks: React, Vue.js, Angular, .NET Core, Laravel
• Unit and integration testing: test writing, code coverage, TDD
• Continuous integration: CI/CD, deployment pipelines
• Ergonomics and accessibility: UX/UI principles, accessibility standards (WCAG)
• MVC architectures: separation of concerns, code organization
• REST APIs: API design, documentation, versioning

Reference knowledge

• Design patterns: Singleton, Factory, Observer, Strategy, etc.
• Automated testing: unit, integration, end-to-end

Reference Situations (Academic Projects and Internship)

• SAÉ 3.01 : Clone Uber (Laravel) - Development of a complete web application with Laravel, MVC architecture, unit testing, integration of complex features. This project allowed me to apply web development principles (R3.01) and development quality (R3.04) with PHP OOP.

• SAÉ 4.01 : Clone Uber (.NET + Vue.js) - Development of a complex application with .NET Core and Vue.js, modern architecture, automated testing, development quality. I used resources R4.01 (Software Architecture) and R4.02 (Development Quality) to create a full-stack application with unit tests and mocks.

• INTERNSHIP : Internship (HEVA) - Professional experience in complex application development, application of best practices, testing and validation. This internship allowed me to put into practice all the skills acquired in a real professional context.

Traces and evidence

Technical architecture : UML diagrams, architecture schemas, detailed technical documentation showing the design of complex applications.

Automated tests : Complete test suites with code coverage reports, demonstrating the ability to guarantee code quality.

CI/CD configurations : GitHub Actions configuration files, Docker Compose, automated deployment scripts.

Structured source code : GitHub repositories with clear architecture, separation of concerns, and complete documentation.

Pedagogical reflection

Level 2 was a turning point in my learning, allowing me to move from implementing isolated features to designing complete and integrated applications. Projects like UberS4 and HEVA particularly taught me to manage technical complexity, integrate different technologies and think in terms of architecture.

The main challenges were managing increasing complexity, integrating multiple technologies, and implementing automated processes. The solutions provided were adopting good architectural practices, using design patterns, and automating testing and deployment processes.
