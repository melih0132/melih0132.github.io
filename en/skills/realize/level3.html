Level definition

Level 3 of the REALIZE competency corresponds to the design and development of advanced applications. This competency represents the third level of learning (Critical Learning AC31), the most advanced level. In my Application Development pathway, only the REALIZE, OPTIMIZE and COLLABORATE competencies are evaluated at level 3 in semesters 5 and 6. At this level, I am able to lead the technical design of a project, optimize performance and security, and ensure system maintenance and evolution.

This level represents my ability to take charge of complex projects from end to end, make strategic technical choices, and guarantee the quality, performance and security of applications in production. These skills are developed during semesters 5 and 6 (third year of BUT Computer Science) through educational resources R5.04, R5.05, R5.06, R5.07, R5.08, R5.09, R5.10, R5.13 and R5.14, as well as academic projects SAÉ 5.01 and my professional work-study experience at Agence Altimax.

Critical Learning (AC) for Level 3

Here are the 3 critical learning items for Level 3:

AC31.01
Choose and implement appropriate architectures
Selection of appropriate architectures according to needs (microservices - architecture with several small independent services, monolithic - application in a single block, distributed - application distributed across several servers, event-driven - event-based architecture), implementation and justification of architectural choices, evaluation of technical constraints (performance, cost, maintainability)
AC31.02
Evolve an existing application
Refactoring (code improvement without changing functionality), migration (transition from one technology to another), architecture evolution, technical debt management (poor quality code to improve), reverse engineering (understanding existing code), code restructuring
AC31.03
Integrate solutions into a production environment
Production deployment (putting the application online for users), monitoring (application surveillance), operational maintenance (bug fixes in production), observability (ability to understand application state)

EVIDENCE 1: SmartHome (Kotlin Application + IoT + AI)

Context

Academic project carried out in the third year (Semester 5) as part of SAÉ 5.01. This is a complex project developing an Android mobile application with Kotlin, integrating IoT (Internet of Things) via MQTT and Raspberry Pi, and using artificial intelligence for data classification.

Trace
  • Kotlin application source code (project in progress, confidential)
  • IoT architecture with MQTT (communication protocol for IoT) and Raspberry Pi
  • AI integration for data classification
  • Technical architecture documentation
Justification

This project allowed me to learn to choose and implement appropriate architectures (AC31.01). I had to think about the best way to combine mobile (Kotlin), IoT (MQTT, Raspberry Pi) and AI to create a coherent system. I also discovered how to integrate a solution into production (AC31.03) by setting up a complete architecture that makes all these technologies work together. I used knowledge from resources R5.05 (Advanced programming - IoT), R5.08 (Kotlin Android), R5.12 (AI - Classification) and R5.09 (Advanced virtualization).

Reflection

This project is the most complex I have carried out so far as it requires the integration of three different domains: mobile development, IoT and artificial intelligence. The main difficulty was understanding how to make these different systems communicate and choosing the most appropriate architecture. I overcame this difficulty by studying IoT architectures, practicing with MQTT, and following AI courses. This project showed me the importance of architectural planning and understanding the technical constraints of each technology.

Context

Professional work-study since September 2025 as a Web Developer at Agence Altimax in Annecy. I develop and maintain websites with WordPress, following project specifications and optimizing performance.

Trace

  • PHP, Twig, Composer, WordPress, JavaScript code for custom features
  • Performance optimizations performed (loading time improvement)

This work-study allowed me to learn to evolve an existing application (AC31.02) by improving and maintaining WordPress sites that are used in production. I resolved bugs and added new features while ensuring I didn't break anything. I also discovered how to integrate a solution into production (AC31.03) by working on real websites, optimizing their performance, and ensuring maintenance so everything continues to work. This daily experience allows me to apply my level 3 skills in a real professional context with all production, performance and maintenance constraints.

Reflection

This work-study is a continuous professional experience that allows me to develop my level 3 skills in a real context. The main difficulty was understanding the existing WordPress architecture and working with code I didn't write. I overcame this difficulty by studying WordPress documentation, analyzing existing code, and communicating regularly with the team. This experience showed me the importance of maintaining and evolving existing applications, as well as the need to understand existing code before modifying it. It also allowed me to develop my ability to optimize performance and solve problems in production.

Challenges encountered
The main challenges were managing complexity at scale, integrating IoT and AI, and maintaining systems in production.
Solutions provided
The solutions provided were adopting good architectural practices and a methodical approach to optimization and maintenance.
