ADMINISTRER - Niveau 1

Apprentissages Critiques (AC13) : Administration de systèmes simples
Première année - Semestres 1 et 2

Retour aux compétences
Définition du niveau

Le Niveau 1 de la compétence ADMINISTRER correspond à l'administration de systèmes simples. Cette compétence représente le premier niveau d'apprentissage (Apprentissages Critiques AC13). À ce niveau, je suis capable de configurer des serveurs de base (serveurs informatiques simples), de gérer les utilisateurs et permissions (droits d'accès), et d'effectuer des sauvegardes (copies de sécurité des données).

Ce niveau représente mes premiers pas dans l'administration de systèmes, où j'ai appris les concepts fondamentaux de la gestion de serveurs et de la configuration de services. Ces compétences ont été développées principalement lors des semestres 1 et 2 (première année de BUT Informatique) à travers les ressources pédagogiques R1.03 et R1.04, ainsi que les projets académiques SAÉ 1.03 et SAÉ 2.03.

Apprentissages Critiques (AC) du Niveau 1

Voici les 3 apprentissages critiques du Niveau 1 :

AC13.01
Installer et configurer un poste de développement
Configuration d'environnements de développement (IDE - Environnement de Développement Intégré, outils de développement), installation d'outils nécessaires, paramétrage de base (réglages initiaux)
AC13.02
Installer des logiciels nécessaires à l'activité
Installation de logiciels (programmes informatiques), gestion des dépendances (logiciels nécessaires pour faire fonctionner d'autres logiciels), configuration des outils
AC13.03
Configurer un environnement de travail simple
Mise en place d'environnements de base (environnements de travail simples), configuration initiale (réglages de départ), paramétrage simple (réglages élémentaires)
Ressources vues en cours (S1-S2)
R1.03 : Introduction à l'architecture des ordinateurs
ÉTUDIÉ
Technologies : Langage C (langage de programmation système)
Contenu : Compréhension de l'architecture matérielle (composants physiques d'un ordinateur), composants d'un ordinateur (processeur, mémoire, stockage), fonctionnement de base. Programmation système en C (programmation proche du matériel).
R1.04 : Introduction aux systèmes d'exploitation et à leur fonctionnement
ÉTUDIÉ
Technologies : Oracle VM VirtualBox (logiciel de virtualisation), Linux (système d'exploitation)
Contenu : Principes des systèmes d'exploitation (logiciels qui gèrent les ressources de l'ordinateur), gestion des processus (programmes en cours d'exécution), fichiers et permissions (droits d'accès aux fichiers), administration de base. Virtualisation avec Oracle VM VirtualBox (création de machines virtuelles), systèmes d'exploitation Linux (utilisation de la ligne de commande).
Contenus abordés

• Installation et configuration d'un poste de développement : IDE, outils, environnement
• Programmation système en C : gestion des processus, fichiers, permissions
• Virtualisation : Oracle VM VirtualBox, création et gestion de machines virtuelles
• Systèmes d'exploitation Linux : gestion des processus, fichiers et permissions, configuration d'environnement de travail simple

Savoirs de référence

• Architecture matérielle : CPU, mémoire, stockage, périphériques
• Systèmes d'exploitation : processus, threads, fichiers, permissions
• Gestion des processus : création, synchronisation, communication
• Fichiers et permissions : droits d'accès, sécurité de base

Situations de référence (Projets académiques)

• SAÉ 1.03 : Machine virtuelle sur Oracle VM VirtualBox - Installation et configuration de machines virtuelles, gestion de systèmes d'exploitation, virtualisation. Ce projet m'a permis d'appliquer les ressources R1.03 (langage C) et R1.04 (VM et Système d'exploitation Linux) pour installer et configurer un poste de développement avec virtualisation. Documentation et configurations disponibles sur GitHub (repository all-my-projects).

• SAÉ 2.03 : Linux sur Raspberry Pi - Installation et configuration de Linux sur Raspberry Pi (mini-ordinateur), administration système, configuration d'environnement. J'ai utilisé les connaissances acquises en systèmes d'exploitation (R1.04) pour configurer un environnement de travail simple sur un système embarqué. Documentation et configurations disponibles sur GitHub (repository all-my-projects).

Traces et preuves contextualisées

Voici mes traces et preuves pour le Niveau 1 :

PREUVE 1 : SAÉ 1.03 - Machine virtuelle sur Oracle VM VirtualBox

Contexte : Projet académique réalisé en première année (Semestre 1) dans le cadre de la SAÉ 1.03. Il s'agissait d'installer et de configurer une machine virtuelle (ordinateur simulé) sur Oracle VM VirtualBox, avec un système d'exploitation Linux, et d'y installer des services web (PHP).

Trace :

  • Configuration de machines virtuelles avec Oracle VM VirtualBox
  • Installation et configuration de Linux sur les machines virtuelles
  • Installation de services web (PHP) sur les machines virtuelles
  • Documentation des configurations et procédures d'installation
  • Fichiers de configuration et scripts disponibles sur GitHub (repository all-my-projects)

Justification : Ce projet m'a permis d'apprendre à installer et configurer un poste de développement (AC13.01) en créant un environnement de développement avec une machine virtuelle. J'ai découvert comment installer les logiciels nécessaires (AC13.02) en installant Linux et les services web sur la machine virtuelle. J'ai aussi appris à configurer un environnement de travail simple (AC13.03) en mettant en place un environnement Linux qui fonctionnait correctement. J'ai utilisé les connaissances des ressources R1.03 (langage C pour comprendre l'architecture) et R1.04 (VM et Linux).

Réflexion : Ce projet m'a permis de comprendre les bases de la virtualisation et de l'administration système. La principale difficulté a été de comprendre comment configurer correctement une machine virtuelle et d'installer Linux dessus. J'ai surmonté cette difficulté en suivant les tutoriels, en pratiquant régulièrement, et en appliquant les concepts enseignés dans les ressources R1.03 et R1.04. Cette expérience m'a montré l'importance de la virtualisation pour créer des environnements de développement isolés et reproductibles.

PREUVE 2 : SAÉ 2.03 - Linux sur Raspberry Pi

Contexte : Projet académique réalisé en première année (Semestre 2) dans le cadre de la SAÉ 2.03. Il s'agissait d'installer et de configurer Linux sur un Raspberry Pi (mini-ordinateur), de configurer l'environnement système, et d'administrer ce système embarqué.

Trace :

  • Installation de Linux sur Raspberry Pi
  • Configuration de l'environnement système (ligne de commande, gestion des processus)
  • Administration système de base (gestion des utilisateurs, permissions, fichiers)
  • Documentation des procédures d'installation et de configuration
  • Fichiers de configuration disponibles sur GitHub (repository all-my-projects)

Justification : Ce projet m'a permis d'apprendre à installer et configurer un poste de développement (AC13.01) en installant Linux sur un Raspberry Pi et en configurant l'environnement pour qu'il fonctionne correctement. J'ai aussi découvert comment installer les logiciels nécessaires (AC13.02) en installant les outils et logiciels dont j'avais besoin sur le système Linux. J'ai appris à configurer un environnement de travail simple (AC13.03) en mettant en place un environnement Linux fonctionnel sur ce système embarqué. J'ai utilisé la ressource R1.04 (Système d'exploitation Linux).

Réflexion : Ce projet a été enrichissant car il m'a permis de travailler sur un système embarqué réel (Raspberry Pi) plutôt que sur une machine virtuelle. La principale difficulté a été de comprendre comment utiliser la ligne de commande Linux pour administrer le système. J'ai surmonté cette difficulté en étudiant les commandes Linux de base, en pratiquant régulièrement, et en appliquant les concepts enseignés dans la ressource R1.04. Cette expérience m'a montré l'importance de l'administration système et m'a préparé à travailler sur des systèmes Linux en production.

Réflexion pédagogique et progression
Défis rencontrés
Les principaux défis à ce niveau concernaient la compréhension de la virtualisation, l'utilisation de la ligne de commande Linux, et la configuration d'environnements système.
Solutions apportées
Pour surmonter ces difficultés, j'ai étudié la documentation des outils (Oracle VM VirtualBox, Linux), j'ai pratiqué régulièrement avec les commandes Linux, et j'ai appliqué les méthodes enseignées en cours.

Progression Ces premières expériences ont été fondamentales pour développer ma compréhension de l'administration système. Elles m'ont préparé à aborder le Niveau 2 avec l'administration de systèmes complexes, le déploiement de services, et la virtualisation avancée avec Docker.
