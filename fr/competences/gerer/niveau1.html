GÉRER - Niveau 1

Apprentissages Critiques (AC14) : Gestion de données simples
Première année - Semestres 1 et 2

Retour aux compétences
Définition du niveau

Le Niveau 1 de la compétence GÉRER correspond à la gestion de données simples. Cette compétence représente le premier niveau d'apprentissage (Apprentissages Critiques AC14). À ce niveau, je suis capable de concevoir des bases de données relationnelles (bases de données organisées en tables liées), d'utiliser SQL (langage de requête pour bases de données) pour manipuler les données, et de gérer l'intégrité des données (garantir la cohérence et la validité des données).

Ce niveau représente mes premiers pas dans la gestion de données, où j'ai appris les concepts fondamentaux des bases de données relationnelles et du langage SQL. Ces compétences ont été développées lors des semestres 1 et 2 (première année de BUT Informatique) à travers la ressource pédagogique R1.05, ainsi que les projets académiques SAÉ 1.04 et SAÉ 2.01.

Composantes Essentielles (CE)

Pour agir avec qualité dans cette compétence, je dois :

  • CE5.01 | Communiquer efficacement avec les différents acteurs d'un projet
  • CE5.02 | Respecter les règles juridiques et les normes en vigueur
  • CE5.03 | Sensibiliser à une gestion éthique, responsable, durable et interculturelle
  • CE5.04 | Adopter une démarche proactive, créative et critique
Situations Professionnelles

Cette compétence s'exerce dans les situations professionnelles suivantes :

  • Lancer un nouveau projet
  • Sécuriser des données
  • Exploiter des données pour la prise de décisions
Apprentissages Critiques (AC) du Niveau 1

Voici les 2 apprentissages critiques du Niveau 1 :

AC14.01
Mettre en place une base de données relationnelle
Conception et création de bases de données relationnelles (organisation en tables), modélisation de schémas de données, mise en place de structures de données cohérentes
AC14.02
Interroger une base de données avec un langage adapté
Utilisation de SQL (langage de requête) pour interroger les bases de données, écriture de requêtes pour récupérer des données, manipulation de données avec un langage adapté
Ressources vues en cours (S1-S2)
R1.05 : SQL (Requêtes, Jointure...)
ÉTUDIÉ
Technologies : SQL (langage de requête pour bases de données)
Contenu : Introduction au modèle relationnel (organisation des données en tables), SQL de base (SELECT, INSERT, UPDATE, DELETE), conception de bases de données simples, requêtes avec jointures (combinaison de données de plusieurs tables).
Contenus abordés

• Conception et mise en place d'une base relationnelle : modélisation, schéma, tables
• Interrogation et mise à jour d'une base de données : SELECT, INSERT, UPDATE, DELETE
• Modèle relationnel : entités, relations, clés primaires et étrangères

Savoirs de référence

• Modèle relationnel : entités, attributs, relations, cardinalités
• SQL de base : SELECT, INSERT, UPDATE, DELETE, WHERE, JOIN
• Normalisation élémentaire : formes normales de base
• Contraintes d'intégrité : clés primaires, clés étrangères, contraintes

Situations de référence (Projets académiques)

• Jeu de données, MCD, MLD, MPD, Script SQL - Conception et mise en place d'une base relationnelle à partir d'un cahier des charges (MCD - Modèle Conceptuel de Données, MLD - Modèle Logique de Données, MPD - Modèle Physique de Données), interrogation et mise à jour d'une base de données (requêtes SQL, jointures). Ce projet m'a permis d'appliquer la ressource R1.05 (SQL : Requêtes, Jointure...) pour créer une base de données complète avec modélisation (MCD, MLD, MPD) et scripts SQL. Projet réalisé dans le cadre de la SAÉ 1.04.

Ce projet m'a permis d'appliquer les compétences de gestion de données en créant une base de données pour la gestion de réservations, avec requêtes SQL et intégrité des données. Projet réalisé dans le cadre de la SAÉ 2.01.

Traces et preuves contextualisées

Voici mes traces et preuves pour le Niveau 1 :

PREUVE 1 : Jeu de données, MCD, MLD, MPD, Script SQL

Contexte

Projet académique réalisé en première année (Semestre 1) dans le cadre de la SAÉ 1.04. Il s'agissait de concevoir et mettre en place une base de données relationnelle complète à partir d'un cahier des charges, en suivant les étapes de modélisation (MCD, MLD, MPD) et en créant les scripts SQL nécessaires.

Trace
  • Modélisation complète : MCD (Modèle Conceptuel de Données), MLD (Modèle Logique de Données), MPD (Modèle Physique de Données)
  • Scripts SQL pour la création de la base de données
  • Requêtes SQL avec jointures pour interroger la base de données
  • Documentation de la structure de la base de données
  • Fiche projet détaillée (voir la fiche projet)
Justification

Ce projet m'a permis d'apprendre à concevoir et mettre en place une base de données relationnelle (AC14.01) en partant d'un cahier des charges. J'ai créé toute la modélisation (MCD, MLD, MPD) avant de passer à l'implémentation avec SQL. J'ai aussi développé mes compétences pour interroger et mettre à jour une base de données (AC14.02) en écrivant des requêtes SQL avec des jointures pour récupérer et manipuler les données efficacement. J'ai utilisé les connaissances acquises dans la ressource R1.05.

Réflexion

Ce projet m'a permis de comprendre l'importance de la modélisation avant l'implémentation. La principale difficulté a été de comprendre comment passer du MCD au MLD puis au MPD, et comment traduire la modélisation en scripts SQL. J'ai surmonté cette difficulté en étudiant les exemples de modélisation, en pratiquant régulièrement, et en appliquant les concepts enseignés dans la ressource R1.05. Cette expérience m'a montré l'importance de la planification et de la modélisation dans la gestion de données.

PREUVE 2 : Dortan App (Base de données relationnelle)

Contexte

Projet académique réalisé en première année (Semestre 2) dans le cadre de la SAÉ 2.01. Il s'agissait de développer une application de gestion de réservations pour la commune de Dortan, nécessitant la création d'une base de données relationnelle intégrée à l'application WPF.

Trace
  • Base de données relationnelle avec tables et relations
  • Requêtes SQL pour interroger et mettre à jour la base de données
  • Intégration de la base de données dans l'application WPF
  • Code source de l'application avec gestion de données disponible sur GitHub
  • Fiche projet détaillée (voir la fiche projet)
Justification

Grâce à ce projet, j'ai pu concevoir et mettre en place une base de données relationnelle (AC14.01) pour gérer les réservations. J'ai créé les tables et défini les relations entre elles pour que tout soit cohérent. J'ai aussi appris à interroger et mettre à jour la base de données (AC14.02) en intégrant des requêtes SQL directement dans l'application WPF pour gérer toutes les opérations sur les réservations. C'était la première fois que j'appliquais mes compétences de gestion de données dans un vrai projet d'application.

Réflexion

Ce projet a été enrichissant car il m'a permis d'intégrer la gestion de données dans une application complète. La principale difficulté a été de comprendre comment connecter l'application WPF à la base de données et comment gérer les opérations de données. J'ai surmonté cette difficulté en étudiant la documentation ADO.NET (technologie pour accéder aux bases de données depuis .NET), en pratiquant avec des exemples, et en appliquant les concepts de SQL appris dans la ressource R1.05. Cette expérience m'a montré l'importance de la gestion de données dans les applications réelles.

Réflexion pédagogique et progression

Le Niveau 1 m'a permis d'acquérir les bases de la gestion de données. À travers des projets comme SAÉ 1.04 (Jeu de données) et Dortan App (SAÉ 2.01), j'ai appris à modéliser des bases de données (MCD, MLD, MPD), à utiliser SQL pour interroger et mettre à jour les données, et à garantir l'intégrité des données.

Défis rencontrés

Les principaux défis à ce niveau concernaient la compréhension de la modélisation relationnelle, la création de requêtes SQL complexes avec jointures, et l'intégration de bases de données dans des applications.

Solutions apportées

Pour surmonter ces difficultés, j'ai étudié les concepts de modélisation relationnelle, j'ai pratiqué régulièrement avec SQL, j'ai créé des bases de données de test, et j'ai appliqué les méthodes enseignées en cours.

Progression Ces premières expériences ont été fondamentales pour développer ma compréhension de la gestion de données. Elles m'ont préparé à aborder le Niveau 2 avec l'optimisation des performances, la sécurité des données, et la visualisation de données avec des outils comme PowerBI.
