Deuxième année - Semestres 3 et 4

Définition du niveau

Le Niveau 2 de la compétence GÉRER correspond à la gestion de données complexes. Cette compétence représente le deuxième niveau d'apprentissage (Apprentissages Critiques AC24). À ce niveau, je suis capable d'optimiser les performances des requêtes (améliorer la vitesse d'exécution), d'utiliser des outils d'analyse comme Power BI (outil de visualisation de données), et de gérer des données volumineuses (grandes quantités de données).

Ce niveau représente une évolution dans ma capacité à analyser et visualiser les données, à optimiser les performances, et à gérer des volumes de données plus importants. Ces compétences ont été développées lors des semestres 3 et 4 (deuxième année de BUT Informatique) à travers les ressources pédagogiques R3.01, R3.07, R3.08, R3.09, R3.10, R3.11, R4.03, R4.06, R4.10 et R4.11, ainsi que les projets académiques SAÉ 2.04, SAÉ 3.01, SAÉ 4.01 et mon stage professionnel chez Helvetics Events.

Composantes Essentielles (CE)

Pour agir avec qualité dans cette compétence, je dois :
• CE5.01 | Communiquer efficacement avec les différents acteurs d'un projet
• CE5.02 | Respecter les règles juridiques et les normes en vigueur

Situations Professionnelles

Cette compétence s'exerce dans les situations professionnelles suivantes :
• Lancer un nouveau projet

• Exploiter des données pour la prise de décisions

Traces et preuves contextualisées

Voici mes traces et preuves pour le Niveau 2 :

PREUVE 1 : Clone Uber (Laravel) - Base de données complexe

Contexte : Projet académique réalisé en deuxième année (Semestre 3) dans le cadre de la SAÉ 3.01. Il s'agissait de développer une application web complète similaire à Uber en utilisant Laravel, nécessitant la gestion d'une base de données complexe avec optimisation des performances et sécurité des données.

Trace :

  • Base de données relationnelle optimisée avec indexation
  • Requêtes SQL optimisées pour les performances
  • Sécurité et intégrité des données implémentées
  • Code source avec gestion de données disponible sur GitHub
  • Fiche projet détaillée (voir la fiche projet)

Justification : Grâce à ce projet, j'ai appris à optimiser les performances d'une base de données (AC24.01) en utilisant l'indexation et en optimisant mes requêtes SQL pour qu'elles soient plus rapides. J'ai aussi découvert l'importance de sécuriser et garantir l'intégrité des données (AC24.02) en mettant en place l'authentification, l'autorisation et des contraintes pour protéger les données. J'ai appliqué les connaissances des ressources R2.06 (Exploitation d'une base de données) et R3.07 (Triggers, Procédures).

Réflexion : Ce projet a été un défi majeur en termes de gestion de données car il fallait gérer de grandes quantités de données (utilisateurs, trajets, calculs) tout en garantissant les performances et la sécurité. La principale difficulté a été d'optimiser les requêtes SQL pour garantir des temps de réponse acceptables. J'ai surmonté cette difficulté en analysant les plans d'exécution des requêtes, en utilisant l'indexation appropriée, et en appliquant les techniques enseignées dans les ressources R2.06 et R3.07. Ce projet m'a montré l'importance de l'optimisation et de la sécurité dans la gestion de données en production.

PREUVE 2 : Clone Uber (.NET + Vue.js) - PostgreSQL avec Code First

Contexte : Projet académique réalisé en deuxième année (Semestre 4) dans le cadre de la SAÉ 4.01. Il s'agissait de développer une application web complète similaire à Uber en utilisant .NET Core pour le backend et Vue.js pour le frontend, avec gestion de données complexes utilisant PostgreSQL avec Code First.

Trace :

  • Base de données relationnelle (PostgreSQL) optimisée avec Code First
  • Modélisation de données avec Entity Framework Core
  • Optimisation des performances de la base de données
  • Sécurité et intégrité des données implémentées
  • Code source avec gestion de données disponible sur GitHub
  • Fiche projet détaillée (voir la fiche projet)

Justification : Ce projet m'a permis d'apprendre à optimiser les performances d'une base de données relationnelle (AC24.01) en utilisant PostgreSQL avec Code First et Entity Framework Core. J'ai découvert comment modéliser les données avec Code First et comment optimiser les requêtes générées par l'ORM. J'ai aussi appris à assurer la sécurité et l'intégrité des données (AC24.02) dans une application .NET Core. J'ai utilisé les ressources R2.06 (Exploitation d'une base de données) et R3.07 (Procédures stockées).

Réflexion : Ce projet a été enrichissant car il m'a permis de travailler avec Entity Framework Core et Code First, une approche moderne de modélisation de données. La principale difficulté a été de comprendre comment optimiser les performances avec un ORM et comment structurer correctement les modèles de données. J'ai surmonté cette difficulté en étudiant les bonnes pratiques d'Entity Framework Core, en optimisant les requêtes LINQ, et en appliquant les concepts enseignés dans les ressources R2.06 et R3.07. Cette expérience m'a montré l'importance de bien comprendre l'ORM pour optimiser les performances de la base de données.

PREUVE 3 : MongoDB - Base de données NoSQL

Contexte : Projet académique réalisé en deuxième année (Semestre 4) dans le cadre de la ressource R4.03 (Qualité et au delà du relationnel - MongoDB). Il s'agissait de maîtriser MongoDB, une base de données NoSQL, à travers des exercices pratiques couvrant les requêtes simples, le framework d'agrégation, les mises à jour, la modélisation, les transactions, la validation JSON Schema, l'indexation et les analyses de performance.

Trace :

  • Repository GitHub avec exercices MongoDB complets : https://github.com/oeoecbien/mongodb
  • Requêtes simples avec comparateurs, opérateurs logiques, projection, regex, tri et pagination
  • Framework d'agrégation avec étapes $match, $project, $unwind, $group, $sort, $lookup
  • Mises à jour avec updateOne, updateMany, opérateurs $set, $unset
  • Modélisation avec duplication de collection, insertion en masse, références simples et enrichies
  • Transactions avec sessions, startTransaction, commitTransaction, abortTransaction
  • Validation JSON Schema avec contraintes required, enum, minimum, maximum, maxItems
  • Indexation avec différents types d'index (simples, composés, texte, multikey, unique, TTL, partiel, sparse, collation)
  • Analyses de performance avec explain("executionStats")

Justification : Ce projet m'a permis d'apprendre à optimiser les performances d'une base de données NoSQL (AC24.01) en maîtrisant MongoDB et ses fonctionnalités d'indexation et d'analyse de performance. J'ai aussi découvert comment assurer la cohérence et la qualité des données (AC24.02) dans une base de données NoSQL en utilisant les transactions et la validation JSON Schema. J'ai utilisé la ressource R4.03 (MongoDB) pour développer une expertise complète en bases de données NoSQL.

Réflexion : Ce projet a été fondamental pour maîtriser MongoDB et comprendre les différences entre bases de données relationnelles et NoSQL. La principale difficulté a été de comprendre les concepts spécifiques à MongoDB comme le framework d'agrégation, les transactions dans un contexte NoSQL, et l'indexation multi-dimensionnelle. J'ai surmonté cette difficulté non seulement en étudiant la documentation officielle, mais aussi en mobilisant les compétences acquises dans les ressources pédagogiques (cœurs), en pratiquant régulièrement avec MongoDB, et en appliquant les concepts enseignés dans la ressource R4.03. Cette expérience m'a montré l'importance de choisir le bon type de base de données selon les besoins du projet et m'a permis de développer une expertise en bases de données NoSQL.

Réflexion pédagogique et progression
Défis rencontrés
Les principaux défis à ce niveau concernaient l'optimisation des performances des requêtes SQL, la gestion de bases de données NoSQL (MongoDB), et la compréhension des différences entre bases de données relationnelles et NoSQL.
Solutions apportées
Pour surmonter ces difficultés, j'ai étudié les techniques d'optimisation SQL, j'ai pratiqué intensivement avec MongoDB à travers des exercices complets, j'ai travaillé sur des projets full-stack utilisant différents types de bases de données, et j'ai appliqué les méthodes enseignées en cours.

Progression Le Niveau 2 m'a permis de développer mes compétences en gestion de données complexes. À travers des projets comme Clone Uber (SAÉ 3.01 et SAÉ 4.01) et MongoDB (R4.03), j'ai appris à optimiser les performances des bases de données relationnelles et NoSQL, à assurer la sécurité et l'intégrité des données, et à maîtriser différents types de bases de données. Ces expériences m'ont permis de développer une expertise en gestion de données complexes, de l'optimisation des performances à la maîtrise des bases de données NoSQL.
