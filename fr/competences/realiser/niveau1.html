RÉALISER - Niveau 1

Apprentissage Critique AC11 : Développement d'applications simples

Retour aux compétences
Définition du niveau

Le Niveau 1 (AC11) de la compétence RÉALISER correspond au développement d'applications simples. À ce niveau, je maîtrise les concepts de base du développement logiciel et je suis capable d'implémenter des fonctionnalités simples selon les spécifications.

Ce niveau représente mes premiers pas dans le développement d'applications, où j'ai appris à utiliser un environnement de développement intégré, à implémenter des fonctionnalités de base et à effectuer des tests unitaires simples. Ces compétences ont été développées principalement lors des semestres 1 et 2 à travers les ressources R1.01, R1.02, R2.01 et R2.02, ainsi que les SAÉ 1.01, 1.02 et 2.01.

Apprentissages Critiques du Niveau 1

• AC11.01 | Implémenter des conceptions simples : Mise en œuvre de conceptions basiques, codage de fonctionnalités simples selon les spécifications, utilisation d'un langage de programmation adapté
• AC11.02 | Élaborer des conceptions simples : Création de conceptions élémentaires, structuration de code simple, modélisation de base
• AC11.03 | Faire des essais et évaluer les résultats au regard des spécifications : Tests simples, validation des fonctionnalités, vérification de conformité aux spécifications, débogage de base
• AC11.04 | Développer des interfaces utilisateurs : Création d'interfaces utilisateur basiques, mise en place de composants UI simples, intégration des interactions utilisateur

Ressources vues en cours (S1-S2)

R1.01 : Initiation au développement → C# (WPF)
Introduction aux concepts fondamentaux de la programmation, structures de contrôle, fonctions, et bases de la programmation. Apprentissage des langages de programmation de base et des outils de développement.

R1.02 : Développement d'interfaces web → HTML, CSS
Création d'interfaces utilisateur web avec HTML, CSS et JavaScript de base. Apprentissage des principes de conception d'interfaces simples, accessibilité de base, et intégration des interactions utilisateur.

R2.01 : Développement orienté objets → Association Liste, Héritage, Abstract, Interface... (C#)
Introduction à la programmation orientée objet : classes, objets, héritage, encapsulation. Structuration de code avec les concepts OOP, modélisation orientée objet de base.

R2.02 : Développement d'applications avec IHM → Kube (Unity)
Développement d'interfaces utilisateur graphiques, gestion des événements, création de composants UI simples, principes d'ergonomie de base.

Contenus abordés

• Programmation C# avec WPF : développement d'applications desktop avec interface graphique
• Programmation orientée objet : héritage, interfaces, classes abstraites, associations, listes
• Développement d'interfaces web : HTML, CSS pour la création d'interfaces utilisateur
• Développement de jeux avec Unity : création d'interfaces utilisateur interactives
• Structures de données : listes, tableaux, dictionnaires
• Gestion des événements : interactions utilisateur, gestion d'état simple

Savoirs de référence

• Langages de programmation : C#, HTML, CSS
• Structures de données : listes, tableaux, dictionnaires
• Programmation orientée objet : héritage, interfaces, classes abstraites, encapsulation
• Conception d'interfaces : WPF, Unity, principes d'ergonomie de base, accessibilité élémentaire

Situations de référence (Projets académiques)

• SAÉ 1.01 : Evit'Shuriken - Développement d'un jeu simple avec Unity, implémentation de mécaniques de base, gestion des interactions utilisateur. Ce projet m'a permis de mettre en pratique les concepts de base du développement (R1.01) et de créer des interfaces utilisateur interactives (R2.02).

• SAÉ 1.02 : Evit'Shuriken - Suite du projet, amélioration des fonctionnalités, tests et validation. Cette phase m'a permis d'approfondir mes compétences en développement et de valider mes connaissances par des tests.

• SAÉ 2.01 : Dortan App - Conception et développement d'une application WPF, programmation orientée objet avec C#, création d'interfaces utilisateur. Ce projet a été l'occasion d'appliquer les concepts de POO (R2.01) avec héritage, interfaces et classes abstraites, ainsi que de développer une interface utilisateur complète avec WPF.

Traces et preuves

Code source commenté : Tous les projets sont disponibles sur GitHub avec des commentaires explicatifs permettant de comprendre l'implémentation des fonctionnalités de base.

Screenshots d'applications : Captures d'écran des interfaces développées montrant la réalisation concrète des fonctionnalités.

Tests unitaires : Fichiers de tests démontrant la capacité à valider le code avec des tests simples.

Documentation technique : Documentation des projets expliquant les choix techniques et l'architecture de base.

Réflexion pédagogique

Le Niveau 1 m'a permis d'acquérir les fondations essentielles du développement logiciel selon le référentiel BUT INFO. À travers des projets concrets comme Evit'Shuriken et Dortan App, j'ai appris à structurer mon code, à utiliser les outils de développement et à implémenter des fonctionnalités selon des spécifications. Ces premières expériences ont été cruciales pour développer ma compréhension des concepts de base et ma confiance en tant que développeur.

Les défis principaux rencontrés à ce niveau concernaient la compréhension des spécifications, la gestion de la complexité du code et l'apprentissage des outils. Les solutions apportées ont été l'utilisation de la documentation, la pratique régulière, la collaboration avec mes pairs, et l'application des méthodes enseignées dans les ressources R1.01 (C# WPF), R1.02 (HTML/CSS), R2.01 (POO C#) et R2.02 (Unity).
