RÉALISER - Niveau 2

Apprentissage Critique AC21 : Développement d'applications complexes

Retour aux compétences
Définition du niveau

Le Niveau 2 (AC21) de la compétence RÉALISER correspond au développement d'applications complexes. À ce niveau, je suis capable de concevoir l'architecture d'une application, d'intégrer plusieurs technologies et frameworks, et de mettre en place des processus de test et de déploiement.

Ce niveau représente une évolution significative dans ma capacité à gérer la complexité technique, à intégrer différentes technologies et à mettre en place des processus automatisés pour garantir la qualité et le déploiement des applications.

Apprentissages Critiques du Niveau 2

• AC21.01 | Élaborer et implémenter les spécifications fonctionnelles et non-fonctionnelles à partir des exigences : Analyse et formalisation des besoins, implémentation des spécifications fonctionnelles et contraintes techniques
• AC21.02 | Appliquer les principes d'ergonomie et d'accessibilité : Conception d'interfaces ergonomiques, respect des standards d'accessibilité, expérience utilisateur optimale
• AC21.03 | Appliquer les bonnes pratiques de conception et de programmation : Patterns de conception, principes SOLID, code propre et maintenable
• AC21.04 | Vérifier et valider une application par les tests : Tests unitaires, tests d'intégration, tests de validation, assurance qualité

Ressources vues en cours (S3-S4)

R2.03 : Qualité de développement → Tests Unitaires
Bonnes pratiques de développement, tests unitaires, intégration continue, qualité du code, maintenabilité.

R3.01 : Développement web → Javascript
Développement d'applications web complètes, architectures MVC, API REST, frameworks modernes, séparation des responsabilités.

R3.04 : Qualité de développement → PHP POO
Approfondissement de la qualité : tests d'intégration, couverture de code, revue de code, assurance qualité, validation et vérification.

R4.01 : Architecture logicielle → WinUI, .NET Core...
Architectures logicielles avancées, patterns de conception, séparation des responsabilités, scalabilité.

R4.02 : Qualité de Développement → NET Core, WinUi (Test Unitaires, Mock...)
Tests unitaires avancés, mocks, qualité du code, maintenabilité, bonnes pratiques de développement.

R4.11 : Développement pour applications mobile → Flutter
Développement d'applications mobiles, interfaces utilisateur adaptatives, gestion d'état, développement cross-platform.

Contenus abordés

• Programmation orientée objet avancée : PHP POO, design patterns, principes SOLID
• Développement web : Javascript, Laravel, architectures MVC
• Tests unitaires et mocks : écriture de tests, couverture de code, utilisation de mocks
• Architectures logicielles : .NET Core, WinUI, séparation des responsabilités
• Développement mobile : Flutter, interfaces utilisateur adaptatives
• Frameworks modernes : Vue.js, .NET Core, Laravel
• Bonnes pratiques de développement : code propre, maintenabilité, qualité

Savoirs de référence

• Design patterns : Singleton, Factory, Observer, Strategy, etc.
• Tests automatisés : unitaires, mocks, couverture de code
• Bonnes pratiques de développement : clean code, refactoring, code review
• Architectures MVC : séparation des responsabilités, organisation du code
• API REST : conception d'APIs, documentation, versioning
• Développement full-stack : intégration frontend/backend, frameworks modernes

Situations de référence (Projets académiques et stage)

• SAÉ 3.01 : Clone Uber (Laravel) - Développement d'une application web complète avec Laravel, architecture MVC, tests unitaires, intégration de fonctionnalités complexes. Ce projet m'a permis d'appliquer les principes de développement web (R3.01) et de qualité de développement (R3.04) avec PHP POO.

• SAÉ 4.01 : Clone Uber (.NET + Vue.js) - Développement d'une application complexe avec .NET Core et Vue.js, architecture moderne, tests automatisés, qualité de développement. J'ai utilisé les ressources R4.01 (Architecture logicielle) et R4.02 (Qualité de développement) pour créer une application full-stack avec tests unitaires et mocks.

• STAGE : Stage (HEVA) - Expérience professionnelle en développement d'applications complexes, application des bonnes pratiques, tests et validation. Ce stage m'a permis de mettre en pratique toutes les compétences acquises dans un contexte professionnel réel.

Traces et preuves

Architecture technique : Diagrammes UML, schémas d'architecture, documentation technique détaillée montrant la conception des applications complexes.

Tests automatisés : Suites de tests complètes avec rapports de couverture de code, démontrant la capacité à garantir la qualité du code.

Configurations CI/CD : Fichiers de configuration GitHub Actions, Docker Compose, scripts de déploiement automatisé.

Code source structuré : Repositories GitHub avec architecture claire, séparation des responsabilités, et documentation complète.

Réflexion pédagogique

Le Niveau 2 a été un tournant dans mon apprentissage, me permettant de passer de la réalisation de fonctionnalités isolées à la conception d'applications complètes et intégrées. Les projets Clone Uber (Laravel et .NET + Vue.js) et mon stage chez HEVA m'ont particulièrement appris à gérer la complexité technique, à intégrer différentes technologies et à penser en termes d'architecture.

Les défis principaux ont été la gestion de la complexité croissante, l'intégration de multiples technologies (Javascript, PHP POO, .NET Core, Vue.js, Flutter), et la mise en place de processus automatisés de test. Les solutions apportées ont été l'adoption de bonnes pratiques d'architecture, l'utilisation de patterns de conception, et l'automatisation des processus de test avec des tests unitaires et des mocks.
